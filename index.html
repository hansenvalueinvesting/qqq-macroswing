How the model works ▸

The idea. A Hidden Markov Model (HMM) assumes the market is always in one of a few invisible "regimes" (here: LONG, SHORT, CHOP), and each regime produces returns with a characteristic mean and volatility. We can't observe the regime directly — only the returns — so we infer it.

The math, in three pieces.

  1. Initial probabilities π — chance the chain starts in each state.
  2. Transition matrix A — chance of moving from state i to state j between bars.
  3. Emission distributions — each state has a Gaussian over [log return, 5d vol] (diagonal covariance, so the two features are treated as conditionally independent given the state).

Training. We run Baum-Welch, an Expectation-Maximization algorithm: (E-step) given current parameters, compute the probability of being in each state on each day via the forward-backward algorithm; (M-step) re-estimate π, A, and the Gaussians using those probabilities as soft assignments. Repeat until log-likelihood stops improving meaningfully. We do this from several random initializations and keep the best fit.

Decoding. The Viterbi algorithm finds the single most likely sequence of regimes (the colored bands on the chart). The probability ribbon shows the softer per-day posteriors.

Labeling. The three trained states are anonymous numbers; we relabel by mean return — highest = LONG, lowest = SHORT, middle = CHOP — so the colors stay consistent across retrainings.

What it is not. The regime is descriptive — it explains what likely happened, not what will happen next. The most recent label can revise as new bars arrive (the model re-smooths the whole history). This is not investment advice.
